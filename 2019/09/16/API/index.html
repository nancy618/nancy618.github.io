API

首先了解一下我们常用的API接口，API是什么呢？

API即应用程序接口（application programming Interface）,是一些预先定义的函数，目的是提供应用程序与开发人员基于某软件或硬件得以访问一组例程的能力，而又无需访问源码，或理解内部工作机制的细节。

操作系统提供的API是由函数、消息、数据结构、数据类型以及语句组成。这是实现的，现成的对象和方法。其主要目的是提高开发效率。

好像API说来说去，就是一种函数。（哈哈~来自小陈的拙见）

基础理论部分，JavaScript：ECMascript （规范语句和语法）、BOM和DOM（响应式布局 ），

1、ECMAScript js的基本语法

2、DOM 文档对象模型，操作的是页面的元素

3、BOM 浏览器对象模型，操作是浏览器

DOM的概念：文档对象模型（Document Object Model，简称DOM），是W3C组织推荐的处理可扩展标志语言的标准编程接口。在网页上，组织页面（或文档）的对象被组织在一个树形结构中，用来表示文档中对象的标准模型就称为DOM。

DOM经常进行的操作：获取元素、动态创建元素、对元素进行操作(设置其属性或调用其方法)。

DOM节点分为标签节点（元素节点）、注释节点、文本节点、属性节点及其他。节点之间的关系就是DOM节点树，可以想象成一棵大树。

页面中的每一个标签都是一个元素

节点（node）：网页中的所有内容都是节点，元素、属性、文本、注释节点，页面中所有的元素都是节点，标签 文本 属性

根节点（root）:html

文档：html文件，一个网页可以称为文档，由于万物皆对象（万物皆可盘， hold住~世界就是你的），这个文档也可以看成一个对象。

元素：网页中的标签

属性：标签的属性

由文档中所有的元素组成的树形结构图，叫做DOM（文档树）。

树上满是分枝，主要包括，Events（事件），Nodes（节点），Ranges（范围操作），Traversal（遍历），Sets（节点集合），而它的每个分枝上结满了果实，接下来详细介绍一下它的果子。

Events（事件）：什么时候做什么事，触发—–响应，大多数 Node.js 核心 API 构建于惯用的异步事件驱动架构，其中某些类型的对象（又称触发器，Emitter）会触发命名事件来调用函数（又称监听器，Listener）。

获取按钮给按钮注册事件 addEventListener( )， 事件名称：click 事件源：谁触发的事件

事件三要素：

事件源:触发(被)事件的元素

事件类型:事件的触发方式(例如鼠标点击或键盘点击)

事件处理程序:事件触发后要执行的代码(函数形式)

事件处理函数：

btn.onclick: function {

​ alert ( ‘….’ );

}

同步函数，异步触发，通过回调函数处理异步。

动态创建元素的三种方式：

doucument.write() 、element.innerHTML 、document.createElement()

appendehild 追加子元素

BOM(Browser Object Model) 是指浏览器对象模型，浏览器对象模型提供了独立于内容的、可以与浏览器窗口进行互动的对象结构。

BOM由多个对象组成，其中代表浏览器窗口的Window （顶级对象），对象是BOM的顶层对象，其他对象都是该对象的子对象。

顶级对象window，window是浏览器的顶级对象，当调用window下的属性和方法时，可以省略window注意：window下一个特殊的属性 window.name。

我们在浏览器中的一些操作都可以使用BOM的方式进行编程处理，比如：刷新浏览器、后退、前进、在浏览器中输入URL（统一资源定位符 Uniform Resource Locator, URL) and so on………

js中加载事件 ：onload 页面加载完成，不过onload 不仅只是Windows的事件！

扯个话进来，关于对话框 【alert() 、prompt() 、confirm()】 ps ：用户体验不好，外观不美观，通常不用来开发。

定时器 : setTimeout() 和 clearTimeout()

setTimeout ( ) 定时炸弹 （隔一段时间执行，仅一次）

clearTimeout ( ) 取消炸弹

setInterval（）闹钟 （隔一段时间执行，会重复）

location对象

location对象是window对象下的一个属性，时候的时候可以省略window对象,location可以获取或者设置浏览器地址栏的URL。
